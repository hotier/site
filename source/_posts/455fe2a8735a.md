# GitHub

## 一、前期准备

配置Git身份（关联GitHub账号）、创建GitHub远程仓库

1. 配置Git身份:

```bash
git config --global user.name "你的GitHub用户名"
git config --global user.email "你的GitHub绑定邮箱"
git config --list   #能看到配置的name和email即可
```

2. GitHub创建远程仓库：
    1. 登录GitHub，点击右上角“+”→`新建 repository`；
    2. 填写仓库名（建议与本地项目名一致），勾选`Public`（公开），无需勾选`Initialize this repository with a README`，点击`Create repository`；
    3. 创建后，复制仓库的HTTPS地址（例：`https://github.com/你的用户名/仓库名.git`），后续关联使用。

## 二、项目上传GitHub（核心命令，仅首次上传）

>进入本地项目根目录（命令：`cd 本地项目路径`，例：`cd Desktop/MyProject`）

1. 初始化本地Git仓库（首次上传必做）：`git init`
2. 将项目所有文件添加到暂存区：`git add .`（小数点不可省略，代表当前目录所有文件）
3. 提交到本地仓库：`git commit -m "首次提交，上传项目完整代码"`（-m后为提交说明，简洁即可）
4. 关联GitHub远程仓库：`git remote add origin 你的GitHub仓库HTTPS地址` ，验证关联：`git remote -v`（显示origin对应的远程地址即成功）
5. 推送本地项目到GitHub（首次推送需加-u）：`git push -u origin main` 说明：GitHub默认主分支为main（部分旧仓库为master），若提示分支不存在，直接推送即可自动创建；首次推送会弹出GitHub登录窗口，输入账号密码（或令牌）即可完成上传。

常见错误：推送失败提示·fatal: `Could not read from remote repository`，检查远程地址是否复制正确，重新执行关联命令即可。

## 三、项目后续更新

核心逻辑：修改代码 → 提交本地 → 推送远程，全程3条核心命令，重复执行即可

1. 修改本地项目代码（开发完成后），执行命令添加到暂存区：`git add .`（若只修改单个文件，可替换为`git add 文件名`）
2. 提交到本地仓库：`git commit -m "更新说明"`（说明需明确，例：`git commit -m "修复登录bug，优化首页样式"`）
3. 推送更新到GitHub（无需加-u，直接推送）：`git push origin main`（分支名与首次推送一致，main/master）

补充命令（更新前必做，避免冲突）：`git pull origin main` 作用：拉取GitHub远程最新代码（若多人协作，他人提交了更新，需先拉取再推送，避免冲突）

简化操作：若提交后发现漏改内容，无需新增提交，执行`git commit --amend`，可修改上一次提交说明或添加漏改文件（先执行`git add 漏改文件`）。

## 四、项目维护（长期使用核心命令）

围绕“仓库整洁、版本可控、冲突解决”，核心命令为主，无需多余描述

### 4.1 版本管理（关键维护）

1. 查看版本记录（排查问题用）：`git log`（完整记录）、`git log --oneline`（简化记录，仅显示版本号和提交说明）
2. 版本回滚（误操作修复）：        - 回滚到上一个版本：`git reset --hard HEAD~1` - 回滚到指定版本：`git reset --hard 版本号`（版本号从git log --oneline中获取）         - 已推送远程的错误版本，回滚本地后强制推送：`git push -f origin main`（需告知协作方，避免覆盖他人代码）
3. 标签管理（发布正式版本用）：        - 创建标签（例：v1.0正式版）：`git tag -a v1.0 -m "正式版本v1.0"` - 推送标签到远程：`git push origin v1.0` - 查看所有标签：`git tag`

### 4.2 仓库清理（优化体积）

1. 本地清理（删除无用缓存、版本对象）：`git gc`
2. 避免提交大文件（核心禁忌）：无需命令，直接不提交视频、压缩包等大文件；若误提交，执行`git rm --cached [大文件名]`，再提交推送即可删除远程大文件。

### 4.3 冲突解决（多人协作维护）

冲突触发：多人修改同一文件同一行，推送时提示冲突，执行以下命令解决

1. 拉取远程冲突代码：`git pull origin main`
2. 打开冲突文件，删除冲突标记（<<<<<<< HEAD、=======、>>>>>>> origin/main），修改为最终需要的内容
3. 重新提交推送：`git add .` → `git commit -m "解决冲突，合并远程更新"` → `git push origin main`

### 4.4 常用维护补充命令

1. 查看远程仓库关联：`git remote -v`
2. 重新关联远程仓库（地址变更用）：`git remote rm origin` → `git remote add origin [新的远程地址]`
3. 恢复误删除的本地文件（已提交过）：`git checkout 文件名`

| 操作场景 | 核心命令 |
